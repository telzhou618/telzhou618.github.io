ssh-gen 命令

举个例子，生成算法为rsa，长度为4096字节的秘钥对。

1

ssh-keygen -t rsa -b 4096 -C "telzhou618@qq.com"

linux 生成本地秘钥

生成 rsa 算法公钥和秘钥,运行ssk-keygen 一路回车，注意，如果秘钥已存在会覆盖

1

ssh-keygen -t rsa -b 4096 -C "telzhou618@qq.com"
  • -t ：指定算法类型
  • -b ：指定秘钥长度
  • -C ：大C，注释说明
  • -f ：保存秘钥的文件名

复制公钥（id_rsa.pub）文件中的内容填写在gitlab中。

然后再 git clone, 不需要再输入用户名和密码。

windows 配置

生成秘钥方法和Linux类似

1
2

ssh-keygen -t rsa -b 4096 -C "telzhou618@qq.com"
cat /c/Users/Administrator/.ssh/id_rsa.pub

设置到 gitlab, 和前面linux一样。

本地多秘钥生成

比如要同时免密访问github和gitlab仓库，就需要生成两组秘钥，入校所示：

  1. 为gitlab生成秘钥,-f指定文件名称
1

ssh-keygen -t rsa -C 'yourEmail@xx.com' -f ~/.ssh/gitlab-rsa
  1. 为github生成秘钥
1

ssh-keygen -t rsa -C 'yourEmail2@xx.com' -f ~/.ssh/github-rsa
  1. 在~/.ssh目录下创建名为config的文件，用于配置不同host的不同ssk key。
1

touch ~/.ssh/config

内容如下所示

# gitlab
Host gitlab.com
    HostName gitlab.com
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/gitlab_id-rsa
# github
Host github.com
    HostName github.com
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/github_id-rsa
    # 配置文件参数
    # Host : Host可以看作是一个你要识别的模式，对识别的模式，进行配置对应的的主机名和ssh文件
    # HostName : 要登录主机的主机名
    # User : 登录名
    # IdentityFile : 指明上面User对应的identityFile路径
  1. 分别在github 和 gitlab 配置公钥信息, 这样就可以再不同平台的代码仓库免密克隆项目了。
